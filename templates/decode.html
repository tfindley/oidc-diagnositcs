    {# ── How to get a JWT ── #}
    How to get a JWT token
    🆕 From this tool

    Log in via the main page, then open the Raw JWT tab on the Claims page. Each token has a Copy button — paste it here.

    The Raw tab shows the ID token, access token, and refresh token in their original dot-separated form, colour-coded by part.

    🔍 Browser DevTools

    Open DevTools (F12), go to the Network tab, and trigger a login. Look for the request to the token endpoint (usually a POST to /token or /callback).

    In the response body you'll find access_token, id_token, and optionally refresh_token as dot-separated strings.

    📄 From an Authorization header

    If your app attaches Bearer tokens to API calls, intercept a request in DevTools (Network → request → Headers) and copy the value after Bearer .

    Alternatively, capture traffic with a proxy like Burp Suite or mitmproxy and extract the token from the Authorization header.

    ⌨ curl (client credentials)

    For server-to-server flows or quick tests, fetch a token directly from the token endpoint:

    curl -s -X POST \
  https://your-provider/token \
  -d grant_type=client_credentials \
  -d client_id=YOUR_CLIENT_ID \
  -d client_secret=YOUR_SECRET \
  | jq -r .access_token

    Replace the URL and credentials with your provider's token endpoint (visible in Provider Metadata on the home page).

    🔒 Keycloak admin console

    In the Keycloak admin UI go to Realm → Sessions, click a user session, then click Show tokens to view raw access and ID tokens.

    You can also use the Evaluate tab under a client's Client scopes section to generate a sample token for any user without a full login flow.

    ⚠ What this decoder does not do

    Signatures are not verified — any three-part dot-separated string will decode. This is intentional for diagnostics: you can inspect tokens from any provider without needing the signing key.

    To verify a token's signature, use the provider's JWKS URI (shown in Provider Metadata) with a tool like jwt.io or a library like python-jose.
